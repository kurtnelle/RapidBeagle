# PocketBeagle Boot Optimization ‚Äî Design Spec

**Date:** 2026-04-25
**Project:** RapidBeagle
**Goal:** Prepare a PocketBeagle Debian image that boots as fast as possible with dotnet as the only first-class user service.

---

## Problem Statement

A PocketBeagle running stock Debian can take up to 30 minutes to become reachable due to:
- BeagleBoard-specific services (cloud9, bonescript, node-red, etc.) blocking boot
- Network-wait units blocking multi-user.target
- Unnecessary kernel modules loading at boot
- Kernel console verbosity (printk) causing I/O overhead
- No visible boot-progress indicator

## Approach

**Aggressive strip to dotnet-appliance baseline (Option B).**

Disable all non-essential services and blacklist non-essential kernel modules. Leave exactly: SSH, the active network stack, USB gadget, USB host (for MIDI controller), ALSA MIDI subsystem, network bridge support, getty on serial, and a pre-written placeholder `dotnet-app.service` unit ready for deployment.

Network type is explicitly not assumed ‚Äî the image may connect via USB gadget, WiFi, or wired ethernet. WiFi drivers are never blacklisted. The dotnet application manages WiFi connections via the NetworkManager D-Bus API.

---

## Script Architecture

### Files

| File | Purpose |
|---|---|
| `optimize-pocketbeagle-boot.sh` | Main script ‚Äî run as root on the device |
| `restore-pocketbeagle-boot.sh` | Generated by `--apply`; reverses all changes |

### Modes (mutually exclusive flags)

| Flag | Behavior |
|---|---|
| `--dry-run` | Prints every action it would take, touches nothing |
| `--apply` | Executes all optimizations, writes restore script, logs all actions |
| `--restore` | Delegates to `restore-pocketbeagle-boot.sh` if present |

### Guard Rails

- Exits immediately if not root
- Exits if run on non-ARM architecture (prevents accidental execution on host)
- All actions logged with timestamp to `/root/boot-diagnostics/optimize-YYYY-MM-DD.log`
- Idempotent ‚Äî checks current state before acting (never double-disables, never double-patches)
- Applied manifest written to `/root/boot-diagnostics/.applied-manifest` ‚Äî restore script reads this

---

## Phase 1 ‚Äî Diagnostic Snapshot

Runs first, regardless of mode. Diagnostic file writes are explicitly permitted in `--dry-run` ‚Äî they are read-only system observations (no service changes, no cmdline patching, no module blacklisting). `--dry-run` prints all *configuration* actions it would take but does not execute them.

Saved to `/boot/firmware/boot-diagnostics/` if the path exists, otherwise to `/root/boot-diagnostics/`.

| File | Command |
|---|---|
| `systemd-analyze.txt` | `systemd-analyze` |
| `systemd-blame.txt` | `systemd-analyze blame` |
| `systemd-critical-chain.txt` | `systemd-analyze critical-chain` |
| `journal-warnings.txt` | `journalctl -b -p warning` |
| `services-enabled.txt` | `systemctl list-unit-files --state=enabled` |
| `services-failed.txt` | `systemctl --failed` |
| `network-wait-status.txt` | Status of all `*-wait-online` units |
| `usb-gadget-status.txt` | Status of `bb-usb-gadgets` and `usb-gadget` |
| `lsmod.txt` | `lsmod` ‚Äî full loaded module list |

---

## Phase 2 ‚Äî Service Disabling (Layer 1)

Uses `systemctl disable --now`. Script checks if each unit file exists before acting. Only units that exist AND are currently enabled will be disabled. Each disabled unit is recorded in `.applied-manifest`.

### Kill List ‚Äî systemd Services

| Category | Services |
|---|---|
| BeagleBoard extras | `cloud9`, `bonescript`, `bonescript-autorun`, `gateone`, `node-red`, `code-server` |
| Web servers | `apache2`, `nginx`, `cockpit`, `lighttpd` |
| mDNS/discovery | `avahi-daemon`, `avahi-dnsconfd` |
| Network blockers | `connman-wait-online`, `NetworkManager-wait-online`, `systemd-networkd-wait-online` |
| Bluetooth | `bluetooth`, `bluealsa`, `btattach`, `hciuart` |
| Printing | `cups`, `cups-browsed` |
| Other non-essential | `ModemManager`, `packagekit`, `snapd`, `apt-daily`, `apt-daily-upgrade` |

### Protected List ‚Äî Never Touched

| Category | Items |
|---|---|
| SSH | `ssh`, `sshd` |
| Networking services | `networking`, `systemd-networkd`, `NetworkManager`, `connman` |
| USB gadget | `bb-usb-gadgets`, `usb-gadget`, `libcomposite`, `usb_f_*`, `g_ether` |
| USB host | `xhci_hcd`, `ehci_hcd`, `ohci_hcd`, `usb_storage` |
| Core system | `systemd-journald`, `systemd-udevd`, `systemd-logind`, `getty@`, `serial-getty@` |

---

## Phase 3 ‚Äî Kernel Module Blacklist (Layer 2)

Written to `/etc/modprobe.d/rapidbeagle-blacklist.conf`. File is written fresh on `--apply`, skipped if already identical (idempotent).

### Blacklisted

| Category | Modules |
|---|---|
| Bluetooth | `btusb`, `bluetooth`, `hci_uart`, `btbcm`, `btintel`, `bnep`, `rfcomm` |
| Audio playback (not MIDI) | `snd_hda_intel`, `snd_hda_codec_generic`, `snd_hda_codec_hdmi`, `snd_pcm`, `snd_soc_core`, `snd_soc_*` |
| Unnecessary HID | *(not blacklisted ‚Äî USB MIDI controllers may use combined HID+MIDI interfaces; blacklisting `usbhid` risks breaking the MIDI controller)* |

### Protected MIDI Modules ‚Äî Never Blacklisted

| Module | Purpose |
|---|---|
| `snd_rawmidi` | Core raw MIDI interface |
| `snd_usbmidi_lib` | USB MIDI transport |
| `snd_usb_audio` | USB audio/MIDI class driver |
| `snd_seq` | ALSA sequencer |
| `snd_seq_midi` | Sequencer MIDI |
| `snd_seq_midi_event` | MIDI event translation |

### Protected Network/Bridge Modules ‚Äî Never Blacklisted

All WiFi drivers (`cfg80211`, `mac80211`, `brcmfmac`, `rtl8*`, `ath*`, `iwlwifi`), bridge (`bridge`, `br_netfilter`, `8021q`), USB host (`xhci_hcd`, `ehci_hcd`).

---

## Phase 4 ‚Äî Kernel Cmdline Optimization (Layer 3)

Suppresses kernel printk console output to reduce I/O overhead during boot.

- Script detects cmdline file: checks `/boot/firmware/extlinux/extlinux.conf` first, then `/boot/cmdline.txt`
- Appends `quiet loglevel=3` to the kernel arguments line
- Backs up original file as `.bak` before patching
- Idempotent: skips if `quiet` is already present
- Restore removes the appended args and restores from `.bak`

---

## Phase 5 ‚Äî NetworkManager No-Auto-Connect Policy

Keeps NetworkManager running (dotnet needs its D-Bus API for WiFi management) but prevents it from autonomously connecting at boot.

- Drops `/etc/NetworkManager/conf.d/99-no-auto.conf` with `no-auto-default=*`
- WiFi drivers remain loaded; NM is present but passive
- dotnet app uses NM D-Bus API (`org.freedesktop.NetworkManager`) to initiate connections
- Restore removes the conf file

---

## Phase 6 ‚Äî USR0 LED Heartbeat Service

**Service:** `pb-heartbeat.service`
**Installed to:** `/etc/systemd/system/`
**LED:** `beaglebone:green:usr0` (reclaimed from kernel `heartbeat` trigger)

### Behavior

| Boot Phase | LED Pattern |
|---|---|
| `sysinit.target` ‚Üí `multi-user.target` | Rapid blink ~5 Hz ("still booting") |
| After `multi-user.target` | Slow blink ~0.5 Hz ("up, app not running") |
| After `dotnet-app.service` active | Steady on ("app running") |

### Implementation

Pure bash loop using `/sys/class/leds/` sysfs interface. No external dependencies. Sets trigger to `none` on start, restores `heartbeat` kernel trigger on stop.

To disable: `systemctl disable pb-heartbeat`

---

## Phase 7 ‚Äî Boot Complete Marker

**Service:** `pb-boot-marker.service`
**Runs after:** `network.target`, `multi-user.target`

Writes to `/boot/firmware/BOOT_COMPLETE.txt` if path exists, otherwise `/root/BOOT_COMPLETE.txt`. Overwrites on every boot.

```
Boot completed: 2026-04-25 09:32:14
Hostname: pocketbeagle
Uptime: 12.4s
IP addresses:
  usb0=192.168.7.2
  eth0=<none>
  wlan0=<none>
Kernel cmdline: ... quiet loglevel=3 ...
```

Readable from host via SCP without an interactive SSH session.

---

## Phase 8 ‚Äî USB Gadget Static IP (Conditional)

Applied only if `usb0` interface exists on the device.

- Writes `/etc/network/interfaces.d/usb0-static` setting `usb0` to `192.168.7.2/24`
- Host expected at `192.168.7.1`
- Skipped if a static config for `usb0` already exists
- Takes effect on next boot (no live restart of networking)
- Restore removes the file

---

## Phase 9 ‚Äî dotnet App Placeholder

**Service:** `dotnet-app.service`
**State:** Installed but disabled

```ini
[Unit]
Description=dotnet Application
After=network.target pb-boot-marker.service

[Service]
Type=notify
ExecStart=/usr/bin/dotnet /opt/app/YourApp.dll
Restart=on-failure
RestartSec=5
User=dotnet

[Install]
WantedBy=multi-user.target
```

Script also creates a `dotnet` system user (no shell, no home dir) so the service slot is ready to use immediately after `dotnet publish`.

To activate: edit `ExecStart`, then `systemctl enable --now dotnet-app`.

---

## Restore Script

Generated by `--apply` at the same path as the main script.

### What it reverses

- Re-enables every service listed in `.applied-manifest` (only those actually disabled by the script)
- Removes `/etc/modprobe.d/rapidbeagle-blacklist.conf`
- Restores original kernel cmdline from `.bak`
- Removes `pb-heartbeat.service` and `pb-boot-marker.service`
- Removes `/etc/NetworkManager/conf.d/99-no-auto.conf`
- Removes `usb0-static` if script created it

### What it does NOT reverse

- Does not remove `dotnet-app.service` (user may have configured it)
- Does not remove the `dotnet` system user
- Does not remove diagnostic log files

## Success Criteria

- Boot to SSH reachable in under 60 seconds (from power-on to first ping reply)
- USR0 LED visibly blinking during boot, slow-blinking after multi-user.target
- `/boot/firmware/BOOT_COMPLETE.txt` present and current after every boot
- `dotnet-app.service` unit present, disabled, ready to activate
- All changes fully reversible via restore script
- Script is idempotent ‚Äî safe to run twice
- `--dry-run` produces a complete action plan with no side effects
